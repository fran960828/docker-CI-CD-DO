### 📝 Introducción a n8n para Agentes de IA

> **Comentario del experto:** Imagina que la IA es un "cerebro" brillante pero que no tiene manos ni acceso a tus herramientas (Gmail, Slack, Calendario). **n8n** actúa como el sistema nervioso que conecta ese cerebro con tus aplicaciones. A diferencia de un simple chat, n8n permite crear flujos de trabajo (workflows) donde la IA puede tomar decisiones, consultar bases de datos y ejecutar acciones automáticamente basándose en eventos específicos.

---

## 1. ¿Qué es n8n? Características, Ventajas y Desventajas

n8n es una herramienta de automatización de flujo de trabajo basada en nodos que permite conectar casi cualquier servicio con una interfaz visual.

- **Ventajas:**
  - **Self-hosted:** Puedes instalarlo en tu propio servidor para máxima privacidad.
  - **Flexibilidad:** Permite escribir código JavaScript personalizado si los nodos estándar no son suficientes.
  - **Costo:** Al ser de código abierto, no pagas por cada ejecución si lo alojas tú mismo.
- **Desventajas:**
  - **Curva de aprendizaje:** Es un poco más técnico que Zapier.
  - **Mantenimiento:** Si lo alojas tú mismo, debes encargarte de las actualizaciones y el servidor.

---

## 2. Integraciones: Zapier vs. Make vs. n8n

Las integraciones son los "puentes" entre aplicaciones. Aquí la comparativa:

- **Zapier:** El más fácil. Ideal para tareas simples de "si pasa esto, haz aquello". Muy caro al escalar.
- **Make (Integromat):** Más visual y potente que Zapier. Permite lógicas complejas pero sigue siendo una plataforma cerrada (SaaS).
- **n8n:** El "pueblo soberano". Ofrece control total sobre los datos, es infinitamente escalable y está optimizado para flujos de IA complejos donde el orden de los datos importa.

---

## 3. Conectividad Total: El Nodo HTTP Request

Si n8n no tiene un nodo oficial para una aplicación (aunque tiene cientos), puedes usar el nodo **HTTP Request**.

- **Para qué sirve:** Permite conectarse a **cualquier API** de internet.
- **Funcionamiento:** Solo necesitas la URL del servicio, el método (GET para pedir datos, POST para enviar) y las credenciales (API Key). Esto asegura que tu agente de IA nunca esté limitado por falta de conectividad.

## 5. Conceptos Clave de la Interfaz

- **Workflows:** El lienzo donde dibujas tu automatización conectando nodos.
- **Credentials:** El lugar seguro donde guardas tus contraseñas y API Keys (OpenAI, Gmail, etc.) para que n8n las use sin que queden expuestas.
- **Executions:** El historial. Aquí ves qué flujos funcionaron, cuáles fallaron y por qué.
- **Admin Panel:** Gestión de usuarios y configuración técnica.
- **Insights:** Gráficas sobre cuánto tiempo y recursos estás ahorrando.
- **Templates:** Flujos ya creados por otros que puedes copiar y pegar.
- **Variables:** Datos que cambian (como el nombre del cliente) y se pasan de un nodo a otro.
- **Help:** Acceso directo a la documentación y el foro de la comunidad.

## 6. Creación de Workflows: El Corazón del Agente

### Workflow 1: Automatización de Correo desde un Chat

Este flujo toma lo que escribes en un chat y lo envía por Gmail.

**Componentes:**

1.  **Trigger (Disparador):** Usaremos el nodo **"Chat Trigger"**. Es la ventana de chat donde escribiremos la orden.
2.  **Gmail Node:** \* **Credentials:** Conecta tu cuenta de Google.
    - **Resource:** Message.
    - **Operation:** Send.
    - **To:** Correo del destinatario.
    - **Subject:** "Mensaje automático desde mi Agente".
    - **Email Type:** Plain Text.
    - **Message:** Aquí usamos una variable. Arrastramos el contenido del **Chat Trigger** al campo "Message".

---

### Workflow 2: Agente de IA con Herramientas (Tooling)

Aquí creamos un cerebro que decide si enviar un correo o no.

**Pasos del Agente:**

1.  **Trigger:** Chat Trigger.
2.  **AI Agent Node:** Es el nodo central.
    - **Prompt/Input:** Viene de la información del Chat.
    - **Model (Cerebro):** Seleccionamos OpenAI (GPT-4o o GPT-4o-mini).
      - _Tip:_ GPT-4o-mini es más rápido y barato; GPT-4o es mejor para razonar.
    - **Memory:** Seleccionamos "Window Buffer Memory" (ponemos un valor de 5 para que recuerde los últimos 5 mensajes).
    - **Tools:** Añadimos el nodo de **Gmail**.
      - _Configuración Crítica:_ En el nodo de Gmail dentro del agente, dejamos los campos (Subject, To, Message) en blanco o configurados para que la IA los rellene dinámicamente basándose en la conversación.

**Ejemplo de uso:**

> **Usuario:** "Dile a juan@empresa.com que llegaré tarde a la reunión por el tráfico".
> **IA:** Entiende la intención, extrae el correo, redacta un asunto profesional y ejecuta el envío.

---

### Workflow 3: Agente de IA con Conexión a Internet

Similar al anterior, pero le damos "ojos" al mundo real.

**Configuración:**

1.  **Trigger:** Chat Trigger.
2.  **AI Agent Node:** Conectado a GPT-4o.
3.  **Tool:** **SERPAPI** (Google Search).
    - Esto permite que si el usuario pregunta algo actual (ej: "¿Cómo está el clima en Madrid hoy?"), la IA no invente la respuesta, sino que use la herramienta de búsqueda para obtener datos reales de Google.

## 🚀 EJEMPLO TÉCNICO COMENTADO (Pseudocódigo de n8n)

```javascript
// PASO 1: TRIGGER
// El usuario escribe: "Busca quién ganó el mundial de F1 y envíaselo a mi jefe"

// PASO 2: EL CEREBRO (AI AGENT)
// Analiza la petición. Detecta que necesita dos cosas:
// 1. Información externa (Usa Tool SERPAPI)
// 2. Enviar un mensaje (Usa Tool GMAIL)

// PASO 3: EJECUCIÓN DE HERRAMIENTAS
// Tool SERPAPI -> Busca "Ganador F1 2024" -> Devuelve "Max Verstappen"
// Tool GMAIL -> Redacta: "Hola, el ganador de la F1 fue Max Verstappen".
//               Destinatario: (Email del jefe configurado previamente)

// PASO 4: RESPUESTA AL CHAT
// IA: "¡Listo! He buscado la información y acabo de enviar el correo a tu jefe."
```
